def fib_bottom_up(n):
    table = [0]*(n+1)
    table[1]=1
    table[2]=1
    for i in range(3,n+1):
        table[i]= table[i-1]+table[i-2]
    return table[n]
def climb_stairs(n):
    if n == 1:
        return 1
    table = [0] * (n + 1)
    table[1] = 1
    table[2] = 2
    for i in range(3, n + 1):
        table[i] = table[i-1] + table[i-2]
    return table[n]

print(climb_stairs(1))    # 1
print(climb_stairs(2))    # 2
print(climb_stairs(3))    # 3
print(climb_stairs(4))    # 5
print(climb_stairs(10))   # 89

NO. That's the entire point of bottom-up DP.
Each value is calculated exactly once and stored in the 
table. When table[4] needs table[3] — it doesn't 
recalculate table[3]. It just reads the stored value. 
Like looking up a dict — instant.

The mental model:
Think of the table like a notepad. You solve a small problem, write the answer down. Next time you need it — you read the notepad. You never solve it again.
That's bottom-up DP. Calculate once. Store. Reuse.
